# Linux real-package owned transport

- Status: hosted transport gate; not public or native release proof
- Date: 2026-07-14
- Parent architecture: [Rust-owned native artifact consumption authority](0003-private-native-artifact-consumption-authority.md)
- Transport dependency: [Linux Rust-owned descriptor transport](0005-linux-owned-descriptor-transport.md)
- Scope: issues #115 and #62

## Decision

Keep the sealed, read-only `memfd` transport selected by issue #139 and test it against real deb and AppImage packages built from the exact checkout under validation. The Linux job now builds both packages before running a fixed Rust integration test. The test discovers only the version-bound Tauri bundle paths derived from `CARGO_MANIFEST_DIR` and `CARGO_PKG_VERSION`; it accepts no caller path, executable, arguments, environment, callback, observation, result, or evidence.

This is an adapter-transport gate. It is deliberately short of #115 acceptance. A pull-request bundle is not an anonymously downloaded public artifact, and a successful hosted command is not `native_proven`.

## Closed probes

The test copies each locally built package into a sealed anonymous memory file, reopens it read-only, validates its device, inode, size, digest, access mode, and seals, and retains the descriptor until the fixed consumer, process group, adopted descendants, and output pipes have settled.

- deb: fixed `/usr/bin/dpkg-deb --extract` consumes only the child-private `/proc/self/fd/198` alias and writes into an internally created mode-700 root. The probe verifies both packaged BatCave binaries, then removes the root. It does not invoke `dpkg --install`, maintainer scripts, package locks, or the package database.
- AppImage: the runtime receives only the inherited owned descriptor and executes its metadata-only `--appimage-offset` operation through three fixed modes: the child-private descriptor path, `execveat(AT_EMPTY_PATH)`, and `fexecve`. The operation executes the AppImage runtime but does not mount the image, extract or launch the BatCave payload, stage an updater, or retain a runtime root.

Every command uses a compiled-in executable or the current fixed test launcher, tokenized arguments, a fixed minimal environment, nonblocking bounded output, a bounded deadline, a new process group, subreaper ownership, `SIGTERM`/`SIGKILL` escalation, adopted-descendant inventory and reaping, and explicit residue cleanup. A descendant that survives the nominal consumer, including one that escapes the original process group while retaining an output pipe, makes the transport fail after bounded cleanup. The exact owned bytes are revalidated after each consumer settles.

## Evidence boundary

The outcome is process-local test state and explicitly records:

- `source_kind: locally_built_bundle`;
- `public_artifact_verified: false`;
- `native_proven: false`; and
- `release_evidence_emitted: false`.

The probe establishes that the selected Linux descriptor transport works with the package tools and runtime generated by this checkout. It does not establish public release identity, anonymous download, checksum or attestation agreement, package trust, installation, launch, runtime identity, settings restart, degradation, telemetry, removal, user-state policy, or #98 evidence.

## Verification

Normal cross-platform Rust validation compiles the test and proves that unsupported hosts do not attempt a Linux fallback. The ignored package probes run only after the hosted Linux job builds and statically verifies both bundles:

```sh
bash scripts/validate-tauri.sh --bundle-only
cargo test --manifest-path src/BatCave.App/src-tauri/Cargo.toml \
  --test linux_package_owned_transport -- --ignored --nocapture
```

Parent issue #115 remains open. Its exact selected public artifact must still be independently re-established by the future Rust-owned complete operation, then passed through the complete ordered install/stage, trust, runtime, removal, settlement, cleanup, and state-policy gates before any native receipt or release evidence can exist.
